# 开发日志 (Dev Logs)

此目录用于存放每次提交或重要里程碑的开发记录，包括遇到的问题、解决方案以及技术决策。

## 命名规范
建议使用以下格式命名文件：
`YYYY-MM-DD_简短描述.md`

例如：
- `2025-11-27_environment_setup_and_troubleshooting.md`
- `2025-11-28_audio_sync_implementation.md`

## 日志模板
每次记录时可参考以下模板：

```markdown
# [标题]

## 1. 变更概述
简要描述本次提交或工作的目标。

## 2. 遇到的问题
- **问题描述**: ...
- **根本原因**: ...
- **解决方案**: ...

## 3. 关键代码/配置变更
...

## 4. 待办/注意事项
...
```
